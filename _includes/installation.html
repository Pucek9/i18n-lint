Installation

CLI

Installing i18n-lint globally via npm gives you the i18n-lint binary.

{% highlight bash %} $ npm install -g i18n-lint {% endhighlight %} {% highlight bash %} $ i18n-lint --help Usage: i18n-lint [OPTIONS] Options: -h, --help output usage information -V, --version output the version number -a, --attributes Comma-separated list of HTML attributes to lint (default: 'alt,title') -i, --ignore-tags Comma-separated list of names of tags to ignore whilst linting (default: 'script,style') -t, --template-delimiters Template delimiters used in source files. For example, Mustache-like templating languages should use '{{,}}' -r, --reporter Specify which reporter to output results with --exclude Comma-separated list of glob patterns to ignore, e.g. "/test_subdir/,ignored.html" --color Force colored output --no-color Disable colored output Use `man i18n-lint` for more information {% endhighlight %}

API

i18n-lint can be used in other projects as a library. After installing, simply require the module.

{% highlight bash %} $ npm install --save i18n-lint {% endhighlight %} {% highlight javascript %} var i18nlint = require('i18n-lint'); // Lint a file var errors = i18nlint('myfile.html', { // ... options ... }); // Lint a string var errors = i18nlint.scan('
Untranslated String!
', { // ...options... {% endhighlight %}

Grunt Plugin

i18n-lint is also available as a grunt plugin: grunt-i18nlint.

{% highlight bash %} $ npm install --save-dev grunt-i18nlint {% endhighlight %} {% highlight javascript %} // Gruntfile.js grunt.loadNpmTasks('grunt-i18nlint'); grunt.initConfig({ // ... snip ... i18nlint: { // ... task config ... } // ... snip ... }); {% endhighlight %}
